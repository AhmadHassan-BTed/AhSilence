# DSP Engine

Technical documentation for the audio processing pipeline.

## FFT Analysis (`FastFourier.kt`)

AhSilence uses a custom **Cooley-Tukey Radix-2 Fast Fourier Transform** to convert time-domain PCM audio into frequency-domain data.

### Algorithm

1. **Bit-reversal permutation** — Reorders the input array in-place for recursive merging
2. **Danielson-Lanczos butterfly operations** — Iteratively computes the DFT using pre-computed cosine/sine tables
3. **Peak detection** — Scans magnitude spectrum (skipping DC at index 0) up to the Nyquist limit

### Zero-Allocation Guarantee

- `real[]` and `imag[]` arrays are pre-allocated at construction
- Trig tables (`cosTable[]`, `sinTable[]`) are computed once in `init`
- `extractDominantFrequency()` reuses the same arrays every call
- No `String`, `List`, or boxed type creation inside the computation

### Frequency Resolution

```
Resolution = sampleRate / bufferSize
           = 44100 / 8192
           ≈ 5.38 Hz per bin
```

This is sufficient for detecting AC hums (typically 50–60 Hz) and electrical drones (100–200 Hz).

## Emission Loop (`NativeAudioDSP.kt`)

### Capture Phase

1. Opens `AudioRecord` at 44.1kHz, mono, PCM 16-bit
2. Waits for the ambient scan duration (default 7 seconds) for environment stabilization
3. Reads one full FFT buffer (8192 samples)
4. Passes buffer to `FastFourier.extractDominantFrequency()`
5. Stores result in `@Volatile detectedFrequency`

### Emission Phase

The GC-free hot loop:

```
while (isRunning && isActive) {
    // 1. Snapshot volatile parameters (once per buffer)
    // 2. Synthesize inverted sine wave into pre-allocated ShortArray
    // 3. Push to AudioTrack with WRITE_BLOCKING
}
```

**Why WRITE_BLOCKING?** It synchronizes the emission loop with the hardware audio clock. Without it, the loop would spin-wait and waste CPU cycles.

### Phase Inversion

The anti-noise wave is generated by applying a phase offset (default 180°) to a sine wave at the detected frequency:

```
sample = sin(currentPhase + phaseOffset) × amplitude × 32767
```

At 180° offset, this produces perfect destructive interference with the original hum — the waves cancel each other mathematically.

## Thread Safety

| Variable | Access Pattern | Mechanism |
|---|---|---|
| `amplitudeFactor` | UI writes, DSP reads | `@Volatile` |
| `phaseOffsetRads` | UI writes, DSP reads | `@Volatile` |
| `detectedFrequency` | DSP writes, UI reads | `@Volatile` |
| `isRunning` | UI writes, DSP reads | `@Volatile` |

All shared state is primitive (`Float`, `Boolean`). `@Volatile` guarantees visibility across threads without locks or GC pressure.
